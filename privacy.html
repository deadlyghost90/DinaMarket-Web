Privacy Policy

Effective Date: June 20, 2026

security Information Security Pledge: We enforce security parameters to safeguard user logins and direct transaction verification structures in accordance with sovereign privacy laws.

1. Data Collection Framework & Scope

DinaMarket Pakistan collects minimal digital telemetry needed to operate peer-to-peer catalog delivery safely. When you use the platform as a Buyer, we process:

  • Authentication Profile Data: Email, Password credentials or verified Google details processed synchronously via Firebase Authentication.
  • Transactional Coordinates: Name, verified cell phone number, and physical Delivery/Pickup locations explicitly provided for checkout sheets.
  • Verification Records: The 12-digit transaction ID (TID) code to verify mobile wallet payments.

2. Merchant Profile Integrity data

When an entrepreneurial user requests Storefront status, we securely store:

  • Business Profile: Storefront Business Name, Description Bio, Store logo URLs, physical address hubs.
  • Communication Coordinates: Real-time WhatsApp contact numbers and discord/social handles required to coordinate self-managed logistics directly with customers.

3. How We Process and Shares Information

Your data is shared exclusively with the direct party involved in the purchase contract to finalize delivery:

  • With Sellers: Buyers’ address, contact phone, name, and TID details are visible to the seller on their orders dashboard to verify EasyPaisa/JazzCash and prepare items.
  • With Buyers: Selected sellers’ public phone, store contact, WhatsApp portal, and transaction details are visible on checkout sheets.

DinaMarket Pakistan will never trade, license, sell, or disclose personal profile data to third-party advertisers or global tracking networks.

4. Live Database Updates and Security Compliance

Through secure Firebase database rules, private data is locked down to authenticated profiles. Delivery logs and direct message notifications are routed directly between the buyer and the verified seller via isolated database paths to block leaks.
